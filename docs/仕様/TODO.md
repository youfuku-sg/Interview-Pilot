# Interview-Pilot TODO

## 0. 現在の作業段階

現在は、まだアプリ本体を編集する段階ではない。

まずは資料を作成し、Pluely の日本語化方針、既存機能、改修方針、ローカル STT / ローカル LLM 構成を整理する。

この段階では、以下を行わない。

- アプリ本体の UI 文言変更
- 機能削除
- ライセンス関連コードの削除
- analytics / telemetry の削除
- データベース変更
- 画面追加
- ローカル STT / ローカル LLM 接続実装

実装に入る前に、日本語化対象一覧と作業順を資料として整理する。

## 1. 目的

この文書は、Interview-Pilot を Pluely ベースの面接支援アプリへ改修していくための大まかな TODO リストである。

詳細な仕様は `要求仕様書.md`、Pluely 既存機能の把握は `Pluely機能把握メモ.md` を参照する。

## 2. 直近の方針

- まずは Pluely の主要 UI を日本語化する
- 日本語化後に、Pluely の既存機能を実際に触って把握する
- 最初の中核機能は、相手側音声の STT と回答方針生成に絞る
- 自分の声の STT は MVP には含めない
- ローカル STT / ローカル LLM 前提で進める
- 個人利用前提で進め、配布や SaaS 化は考えない
