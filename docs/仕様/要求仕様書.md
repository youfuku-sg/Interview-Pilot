# 面接支援アプリ 要求仕様書

## 1. 文書の目的

この文書は、Pluely をベースにした面接支援アプリを、今後どの方向へ改修していくかを整理するための要求仕様書である。

本書では、初期開発で実現する MVP、将来的に拡張する機能、現時点で決め切らない論点を分けて記載する。

本アプリは、当面は開発者本人が個人利用することを前提とする。第三者への配布、SaaS 化、商用提供は現時点では想定しない。

## 2. プロダクト概要

本アプリは、転職面接の準備、面接中の質問理解、面接後の振り返りを支援するデスクトップアプリである。

面接中に長文の回答を代筆することではなく、応募者が自分の経験を思い出し、質問意図に沿って回答を組み立てるための短い補助情報を提示することを主目的とする。

音声認識と回答生成は、ローカル STT とローカル LLM の利用を基本方針とする。外部 AI API は初期構想の主経路には含めず、必要になった場合の拡張候補として扱う。

## 3. 背景と課題

転職面接では、応募者が自身の経験や強みを十分に持っていても、緊張や質問意図の読み違いにより、適切に回答できないことがある。

特にエンジニア面接では、過去の技術経験、意思決定、トラブル対応、チーム開発、キャリア観など、複数の観点から深掘りされる。事前に準備した職務経歴書や求人票の情報を、面接中に瞬時に思い出すことは難しい。

このアプリは、事前に登録した資料をもとに、面接中の質問に対して以下を短時間で提示する。

- 質問の意図
- 回答で触れるべき要点
- 関連する職務経歴・実績
- 避けた方がよい回答方針
- 面接後に見返すための質問履歴

## 4. 想定ユーザー

- 転職活動中のエンジニア
- 面接に苦手意識がある応募者
- 職務経歴をうまく言語化したい応募者
- 面接前後の準備と振り返りを効率化したい応募者

## 5. 利用シーン

### 5.1 面接前

ユーザーは、職務経歴書、自己 PR、志望動機、求人票、応募企業情報などを登録する。

アプリは登録資料を整理し、面接で使えそうな経験、技術、実績、想定質問を確認できる状態にする。

### 5.2 面接中

ユーザーは、面接画面の横や上に小さなオーバーレイを表示する。

アプリは Web 会議などの相手側音声を文字起こしし、面接官の質問意図と回答方針を短く提示する。

提示内容は、読み上げるための台本ではなく、回答を組み立てるための補助メモを中心とする。

### 5.3 面接後

ユーザーは、面接中に出た質問、AI が提示した回答方針、自分で残したメモを見返す。

アプリは、次回までに準備すべき回答、職務経歴書に追記した方がよい内容、苦手な質問カテゴリを整理する。

## 6. スコープ

### 6.1 MVP で実現する範囲

- 職務経歴書のテキスト登録
- 求人票のテキスト登録

- 登録資料を参照した回答方針の提示

- ローカル SQLite への履歴保存

- 企業向け管理画面

- 自分の発話音声の文字起こし
- 自分の回答内容の自動採点

## 7. 機能要求

### 7.1 事前資料管理

ユーザーは、面接で参照したい資料を登録、編集、削除できる。

登録できる資料種別は以下とする。

- 職務経歴書
- 自己 PR
- 志望動機
- 求人票
- 応募企業情報
- 想定質問と回答メモ

MVP では、資料本文はテキスト入力またはテキスト貼り付けを前提とする。

各資料には、以下の情報を保存する。

- 資料 ID
- 資料種別
- タイトル
- 本文
- 関連する応募先または面接セッション
- 作成日時
- 更新日時

### 7.2 応募先管理

ユーザーは、応募先企業または求人ごとに情報を管理できる。

MVP では、以下の項目を扱う。

- 企業名
- 職種名
- 求人票本文
- 志望動機メモ
- 面接メモ
- 関連資料

応募先情報は、面接中の回答支援で参照される。

### 7.3 面接セッション管理

ユーザーは、面接ごとにセッションを作成できる。

セッションには以下の情報を保存する。

- セッション ID
- 応募先
- 面接名
- 面接日時
- 面接種別
- 文字起こしログ
- 抽出された質問
- AI が提示した回答方針
- ユーザーメモ
- セッション状態

面接種別は、一次面接、技術面接、最終面接、カジュアル面談、その他を想定する。

### 7.4 文字起こし

アプリは、面接中の相手側音声を文字起こしできる。

MVP では、既存 Pluely のシステム音声取得機能を主に活用する。

自分のマイク音声を文字起こしすることは、MVP の主要件には含めない。ユーザーが任意でメモを残すための手入力や将来的な音声メモは拡張候補として扱う。

文字起こし結果は、時系列の発話ログとして保存する。

発話ログには以下を含める。

- 発話 ID
- セッション ID
- 発話テキスト
- 取得元
- 開始時刻
- 終了時刻
- 質問候補かどうか

取得元は、システム音声、手入力のいずれかを想定する。

### 7.5 質問抽出

アプリは、文字起こしされた発話の中から、面接官の質問らしい発話を抽出する。

MVP では、以下のいずれかまたは組み合わせで質問候補を判定する。

- 文末表現による簡易判定
- 疑問詞による判定
- AI による質問判定
- ユーザーによる手動選択

誤抽出を前提とし、ユーザーが質問候補を削除または編集できるようにする。

### 7.6 質問意図分類

アプリは、抽出した質問をカテゴリ分類する。

分類カテゴリは以下を初期値とする。

- 経歴確認
- 技術経験
- 技術深掘り
- 転職理由
- 志望動機
- 強み
- 弱み
- チーム開発
- トラブル対応
- マネジメント経験
- キャリアプラン
- 条件確認
- 逆質問
- その他

分類結果は、回答支援と面接後レポートで利用する。

### 7.7 回答支援

アプリは、質問、質問意図、登録資料をもとに回答方針を生成する。

面接中の表示は、長文ではなく短い構造化メモを基本とする。

回答支援の出力項目は以下とする。

- 質問意図
- 回答の方向性
- 話すべき要点
- 関連する実績または経験
- 避けた方がよい表現
- 必要に応じた 30 秒回答例

MVP では、60 秒以上の長文回答例は優先しない。

回答支援は、ユーザーの登録資料に存在しない経歴や実績を断定的に作らない。

### 7.8 面接中オーバーレイ

アプリは、面接中に常時表示できる小型のオーバーレイ画面を提供する。

オーバーレイに表示する情報は以下とする。

- 最新の質問
- 質問意図
- 回答方針
- 話すべき要点
- 関連資料の短い引用または要約
- 処理状態

オーバーレイは、面接中に視線移動が少なく済むよう、情報量を抑える。

表示文は、箇条書き中心とし、1 項目あたり短文を基本とする。

### 7.9 面接後レポート

アプリは、面接終了後にセッション内容を振り返る画面を提供する。

面接後レポートには以下を表示する。

- 面接中に出た質問一覧
- 各質問の意図分類
- AI が提示した回答方針
- ユーザーメモ
- うまく答えられなかった質問
- 次回までの準備項目
- 職務経歴書に追記するとよい内容

MVP では PDF 出力は対象外とし、画面表示と Markdown 相当のテキストコピーを優先する。

### 7.10 AI プロバイダ設定

アプリは、AI 回答生成および音声認識に使うローカルプロバイダを設定できる。

MVP では、既存 Pluely のプロバイダ設定を活用しつつ、以下を優先する。

- Ollama または LM Studio などのローカル LLM
- Whisper、faster-whisper、whisper.cpp などのローカル STT
- OpenAI 互換 API 形式で接続できるローカル推論サーバー

OpenAI Speech to Text やクラウド LLM などの外部 API は、MVP の主要要件には含めない。

API キーは安全に保存する。

外部 API 利用を将来追加する場合は、どの情報が送信される可能性があるかをユーザーが理解できる表示を用意する。

### 7.11 データ保存

アプリは、ユーザーの資料、面接セッション、質問履歴、AI 出力をローカルに保存する。

MVP では SQLite を利用する。

保存対象は以下とする。

- 事前資料
- 応募先情報
- 面接セッション
- 文字起こしログ
- 質問一覧
- 回答支援結果
- ユーザーメモ
- 設定値

ユーザーは、保存されたセッションを削除できる。

## 8. 非機能要求

### 8.1 対応環境

MVP の主対象は Windows デスクトップとする。

Tauri の既存構成を活かし、macOS と Linux も可能な範囲で維持する。ただし、音声取得やオーバーレイ挙動は OS ごとの差分を許容する。

### 8.2 応答速度

面接中の回答支援は、質問確定から 5〜10 秒以内に初回の方針を表示することを目標とする。

AI 応答が遅い場合でも、処理中であることが分かる表示を出す。

### 8.3 UI と認知負荷

面接中 UI は、短時間で読めることを最優先する。

以下を避ける。

- 長文の回答生成をそのまま表示すること
- 複数のカードや装飾で視線移動が増えること
- 面接中に設定操作が必要になること
- 重要情報がスクロールしないと読めないこと

### 8.4 プライバシー

職務経歴書、求人票、面接内容は個人情報または機密情報を含む可能性がある。

アプリは、ローカル保存とローカル処理を基本とする。

MVP では、音声認識と回答生成のために面接内容や職務経歴情報を外部 AI API に送信しないことを原則とする。

外部 API を将来利用する場合は、どの情報が送信される可能性があるかをユーザーに明示する。

不要な telemetry、analytics、外部送信は削除または無効化する。

### 8.5 セキュリティ

API キーなどの秘密情報は、平文の設定ファイルに保存しない。

可能な限り OS のキーチェーンまたは既存の secure storage 機構を利用する。

面接セッションと資料は、ユーザー操作で削除できるようにする。

### 8.6 倫理と利用方針

本アプリは、応募者が自分の経験を整理して回答するための補助ツールとする。

以下の方向性は避ける。

- 経歴や実績の捏造を助長すること
- 面接官を欺くことを前面に出した表現
- 「検知されない」「隠れて使える」ことを価値として訴求すること
- ユーザー本人が理解していない回答を丸読みさせること

アプリ内の文言も、面接準備と回答補助の方向に統一する。

### 8.7 ライセンス

本プロジェクトは Pluely をベースとしており、GPL-3.0 の影響を受ける。

当面は開発者本人の個人利用のみを想定するため、改修版の第三者配布は行わない。

個人利用や検証用途では改修しやすいが、配布や商用化を行う場合は GPL-3.0 に従う必要がある。

将来的にクローズドな商用製品化を検討する場合は、ベース実装の再選定またはライセンス整理を別途行う。

資料管理画面では、職務経歴書、求人票、自己 PR、企業情報を登録、編集、削除できる。

面接セッション画面では、面接の開始、停止、文字起こしログ、質問一覧、回答支援結果を確認できる。

オーバーレイ画面では、面接中に必要な最新情報だけを表示する。

### 9.6 振り返り画面

振り返り画面では、面接後に質問一覧、回答方針、改善点を確認できる。

### 9.7 設定画面

設定画面では、ローカル LLM、ローカル STT、ショートカット、保存設定、プライバシー設定を管理できる。

## 11. AI 処理フロー

MVP の基本フローは以下とする。

1. システム音声から相手側発話を文字起こしする
2. 発話ログを保存する
3. 質問候補を抽出する
4. 質問意図を分類する
5. 関連する登録資料を検索する
6. 回答支援プロンプトを組み立てる
7. AI 応答を生成する
8. オーバーレイに短く表示する
9. セッション履歴に保存する

MVP では、関連資料検索は単純なキーワード検索または全文検索から始める。

ベクトル検索や本格的な RAG は、資料量や精度課題が明確になってから導入を検討する。

## 12. 受け入れ基準

MVP の受け入れ基準は以下とする。

- ユーザーが職務経歴書と求人票を登録できる
- ユーザーが面接セッションを開始できる
- システム音声から文字起こしログが蓄積される
- 文字起こしログから質問候補が作成される
- 質問候補に対して意図分類が表示される
- 登録資料を踏まえた回答方針が表示される
- 回答方針は面接中に読める短い形式で表示される
- 面接終了後に質問一覧を見返せる
- セッション履歴を削除できる
- 不要な外部 analytics が送信されない
- 音声認識と回答生成がローカル STT / ローカル LLM を使って動作する
