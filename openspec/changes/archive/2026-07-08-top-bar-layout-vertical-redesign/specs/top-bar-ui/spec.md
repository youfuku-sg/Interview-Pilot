## MODIFIED Requirements

### Requirement: Top bar layout structure
トップバーは左カラム（アイコン縦並び）と右エリア（テキスト入力）の2カラム構成とする。左カラムの幅は固定（約40px）、右エリアは残り幅をすべて占める。トップバーの collapsed 時の縦幅は約160pxとし、現状（54px）の約3倍とする。

#### Scenario: 通常時（未キャプチャ）のレイアウトが2カラムである
- **WHEN** アプリが起動し、音声キャプチャが停止中の状態でトップバーが表示される
- **THEN** 左カラムにアイコンボタン群（SystemAudioボタン・マイクボタン・設定ボタン・ドラッグハンドル）が縦に並んで表示される
- **AND** 右エリアにテキスト入力欄が表示される

#### Scenario: 左カラムのアイコンが縦に並ぶ
- **WHEN** トップバーが表示される
- **THEN** SystemAudioボタン、マイクボタン、設定ボタン、ドラッグハンドルが上から順に縦方向に配置される

#### Scenario: トップバーの collapsed 縦幅が約160pxである
- **WHEN** ポップオーバーが閉じている状態（collapsed）でトップバーが表示される
- **THEN** ウィンドウの高さが約160pxである

#### Scenario: 音声キャプチャ中も2カラム構成を維持する
- **WHEN** システム音声のキャプチャが開始される
- **THEN** 左カラムにステータス表示、右エリアにオーディオビジュアライザーが表示される
- **AND** 2カラムのレイアウト構造が維持される

#### Scenario: テキスト入力欄が右エリアの幅いっぱいに広がる
- **WHEN** トップバーが表示される
- **THEN** テキスト入力欄が右エリアの横幅を占有して表示される
