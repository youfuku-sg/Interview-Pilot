## Context

`top-bar-layout-vertical-redesign`（②）実装後の右エリアは `flex-1 flex flex-col justify-center` で `Input` を縦中央に置く構造。これを3行に分割する。ウィンドウ collapsed 高さは 160px（②で設定済み）。

目標の右エリア内部構造:

```
┌─────────────────────────────────┐
│  上段（flex-1）  空白エリアA     │  ← 将来用
├─────────────────────────────────┤
│  中段（flex-1）  空白エリアB     │  ← 将来用
├─────────────────────────────────┤
│  下段（auto）    テキスト入力欄  │  ← Input コンポーネント
└─────────────────────────────────┘
```

## Goals / Non-Goals

**Goals:**
- 右エリアを `flex-col` で3段に分割する
- 上段・中段は `flex-1` で均等に残りの高さを占有する（空の `div` でよい）
- 下段は `Input` コンポーネントを配置し、高さは入力欄のコンテンツに合わせる（`shrink-0`）
- 境界線（`border-b`）を上段・中段の間に入れてエリアを視覚的に区切る

**Non-Goals:**
- 上段・中段にコンテンツを実装することはこの change では行わない
- 3段の高さ比率を細かく指定する設定UIは作らない

## Decisions

### 高さ配分: `flex-1 / flex-1 / shrink-0`

下段（入力欄）の高さをコンテンツ固定にして、残りを上中で均等に分けるのが最も自然。`flex-1` × 2 + `shrink-0` 構成とする。

### 上段・中段の空白スタイル

将来コンテンツを挿入しやすいよう `div` に `data-slot="top-panel"` / `data-slot="middle-panel"` を付与してセマンティクスを持たせる。現状は背景色なし・ボーダーのみ。

### 境界線

`border-b border-border/40` を上段・中段の下端に追加して3つのエリアを明示する。下段にはボーダーなし（Cardの外枠で十分）。

## Risks / Trade-offs

- **リスク: 右エリアの高さが足りない場合** → collapsed 160px のうち左カラムのパディングを除くと右エリアは約144px。3分割すると上中各約48px、下段が入力欄高さ（約36px）になり十分収まる。
- **トレードオフ: 上段・中段が空白** → 一時的に余白が目立つが、将来のコンテンツを追加するための意図的な確保。プレースホルダーテキストは入れない。
