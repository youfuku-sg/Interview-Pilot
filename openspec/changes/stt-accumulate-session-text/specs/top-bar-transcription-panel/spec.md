## MODIFIED Requirements

### Requirement: 文字起こしパネルの表示
トップバー右エリアの上段は文字起こしパネルとして機能し、`useSystemAudio` フックが提供する `sessionTranscript`（セッション中の全発話の配列）をリアルタイムで表示しなければならない（SHALL）。各発話は改行で区切られて縦に並んで表示される。

#### Scenario: STTプロバイダー未設定時にエラーを表示する
- **WHEN** STTプロバイダーが設定されておらず、かつ Pluely API が無効の状態でトップバーが表示される
- **THEN** 上段に「STTプロバイダーが選択されていません」というエラーメッセージが表示される

#### Scenario: 音声処理中にローディングを表示する
- **WHEN** 音声キャプチャが進行中で文字起こし処理（`isProcessing`）が実行されている
- **THEN** 上段にローディングインジケーターが表示される

#### Scenario: 1回目の発話後にテキストを表示する
- **WHEN** 最初の発話が認識されて文字起こしが完了する
- **THEN** 上段に1行目として文字起こしされたテキストが表示される

#### Scenario: 2回目以降の発話が改行で追加される
- **WHEN** 同一セッション中に2回目以降の発話が認識されて文字起こしが完了する
- **THEN** 上段に既存のテキストを残したまま新しい発話が改行されて追加表示される

#### Scenario: 文字起こし結果がない場合は空白を表示する
- **WHEN** `sessionTranscript` が空で、STTプロバイダーが設定済みで、処理中でもない
- **THEN** 上段は空白（何も表示しない）で表示される
