## ADDED Requirements

### Requirement: セッション累積発話をAIプロバイダーで要約する
システムは、`sessionTranscript`（セッション中に累積された発話配列）を対象に、設定済みのAIプロバイダーを使って要約テキストを生成しなければならない（SHALL）。要約は面接回答生成用のシステムプロンプトとは別の、要約専用のシステムプロンプトを用いて生成する。

#### Scenario: 新しい発話が確定するたびに要約を再生成する
- **WHEN** 音声キャプチャ中に新しい発話が文字起こしされ`sessionTranscript`に追加される
- **THEN** その時点までの`sessionTranscript`全体を入力として要約リクエストが送信され、要約テキストが再生成される

#### Scenario: 進行中の要約リクエストは新しい発話が来ると中断される
- **WHEN** 要約生成が完了する前に次の発話が確定する
- **THEN** 進行中の要約リクエストは中断され、最新の`sessionTranscript`を対象とした要約リクエストが新たに開始される

#### Scenario: AIプロバイダー未設定時は要約を生成しない
- **WHEN** AIプロバイダーが設定されておらず、かつ Pluely API が無効の状態で発話が確定する
- **THEN** 要約リクエストは送信されず、エラーも表示されない

### Requirement: トップバー中段に要約パネルとして表示する
トップバー右エリアの中段は、要約パネルとして機能しなければならない（SHALL）。要約が存在する場合はそのテキストを表示し、要約生成中はローディング表示を、AIプロバイダー未設定時は空欄を表示する。

#### Scenario: 要約テキストが中段に表示される
- **WHEN** 要約生成が完了し要約テキストが存在する
- **THEN** 中段に要約テキストが表示される

#### Scenario: 要約生成中はローディングインジケーターを表示する
- **WHEN** 要約リクエストが進行中である
- **THEN** 中段にローディングインジケーターが表示される

#### Scenario: AIプロバイダー未設定時に中段は空欄である
- **WHEN** AIプロバイダーが設定されておらず、かつ Pluely API が無効の状態でトップバーが表示される
- **THEN** 中段には何も表示されない（エラーメッセージは表示しない）

### Requirement: セッションリセット時に要約状態をクリアする
セッションが再開始される（新規キャプチャ開始または新規会話開始）とき、システムは要約テキストと要約生成中フラグをクリアしなければならない（SHALL）。

#### Scenario: 新規キャプチャ開始時に前セッションの要約が残らない
- **WHEN** ユーザーがシステム音声キャプチャを新規に開始する
- **THEN** 中段の要約表示が空の状態から始まる

#### Scenario: 新規会話開始時に前セッションの要約が残らない
- **WHEN** ユーザーが新規会話を開始する
- **THEN** 中段の要約表示が空の状態から始まる
