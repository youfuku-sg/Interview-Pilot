## MODIFIED Requirements

### Requirement: Top bar layout structure
トップバーは左カラム（アイコン縦並び）と右エリア（文字起こし・要約・AI回答を表示する領域）の2カラム構成でなければならない（SHALL）。左カラムの幅は固定（約40px）、右エリアは残り幅をすべて占める。トップバーの collapsed 時の縦幅は約240pxとし、現状（160px）の1.5倍とする。左カラムの配置順は上から SystemAudioボタン・マイクボタン・設定ボタン・終了ボタン・ドラッグハンドルとする。テキスト入力欄はデフォルトで非表示とする。

#### Scenario: 通常時（未キャプチャ）のレイアウトが2カラムである
- **WHEN** アプリが起動し、音声キャプチャが停止中の状態でトップバーが表示される
- **THEN** 左カラムにアイコンボタン群（SystemAudioボタン・マイクボタン・設定ボタン・終了ボタン・ドラッグハンドル）が縦に並んで表示される
- **AND** 右エリアに文字起こしパネル・要約パネル・AI回答パネルが表示される

#### Scenario: 左カラムのアイコンが正しい順序で縦に並ぶ
- **WHEN** トップバーが表示される
- **THEN** 上から順に SystemAudioボタン、マイクボタン、設定ボタン、終了ボタン、ドラッグハンドルが縦方向に配置される

#### Scenario: トップバーの collapsed 縦幅が約240pxである
- **WHEN** ポップオーバーが閉じている状態（collapsed）でトップバーが表示される
- **THEN** ウィンドウの高さが約240pxである

#### Scenario: 音声キャプチャ中も2カラム構成を維持する
- **WHEN** システム音声のキャプチャが開始される
- **THEN** 左カラムにステータス表示、右エリアにオーディオビジュアライザーが表示される
- **AND** 2カラムのレイアウト構造が維持される

#### Scenario: テキスト入力欄はデフォルトで非表示である
- **WHEN** トップバーが表示される
- **THEN** 右エリアにテキスト入力欄は表示されない

#### Scenario: 終了ボタンを押すとアプリが終了する
- **WHEN** ユーザーが左カラムの終了ボタンを押す
- **THEN** アプリケーションプロセスが終了する

### Requirement: Top bar right panel layout
トップバーの右エリアは縦に3段（上段・中段・下段）に分割されなければならない（SHALL）。上段は文字起こしパネル、中段は要約パネル（セッション累積発話をAIプロバイダーで要約したテキストを表示。AIプロバイダーが設定されていない場合は空欄）、下段はAI回答パネル（直近発話へのAI回答を表示。AIプロバイダーが設定されていない場合は空欄）として機能する。

#### Scenario: 右エリアが3段に分割されている
- **WHEN** アプリが起動し、トップバーが表示される
- **THEN** 右エリアが上段・中段・下段の3つの横枠に縦分割されている

#### Scenario: 上段が文字起こしパネルとして機能する
- **WHEN** 音声キャプチャが有効でトップバーが表示される
- **THEN** 上段に累積された文字起こし結果が表示される

#### Scenario: 中段が要約パネルとして機能する
- **WHEN** 文字起こし後に要約処理が完了する
- **THEN** 中段にセッション累積発話の要約テキストが表示される

#### Scenario: 下段がAI回答パネルとして機能する
- **WHEN** 文字起こし後にAI処理が完了する
- **THEN** 下段にAIの回答テキストが表示される

#### Scenario: AIプロバイダー未設定時に中段・下段は空欄である
- **WHEN** AIプロバイダーが設定されておらず、かつ Pluely API が無効の状態でトップバーが表示される
- **THEN** 中段・下段には何も表示されない（エラーメッセージは表示しない）

#### Scenario: AI処理中にローディングを表示する
- **WHEN** 文字起こし完了後に要約処理またはAI処理（`isAIProcessing`）が実行されている
- **THEN** 対応する段（中段は要約処理中、下段はAI処理中）にローディングインジケーターが表示される

#### Scenario: 上段・中段・下段の高さが均等に分配される
- **WHEN** トップバーが表示される
- **THEN** 上段・中段・下段がそれぞれ利用可能な高さの3分の1ずつを占める

#### Scenario: 各段の境界が視覚的に区別できる
- **WHEN** トップバーが表示される
- **THEN** 上段・中段・下段の境界線が表示され、3つのエリアが視覚的に区別できる
