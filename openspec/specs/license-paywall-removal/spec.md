## Purpose

このフォークにはライセンス購入・決済バックエンドが存在せず、`hasActiveLicense` が正当に `true` になることは決してないため、フロントエンドのライセンスゲート(機能ロック・購入導線)を撤去し、全機能を常に利用可能にするための要件を定義する。システムレベル(Rust/Tauri)のライセンスゲートはこの capability の対象外であり、明示的に変更しない。

## Requirements

### Requirement: これまでライセンスでゲートされていた製品機能は常に利用可能である
`hasActiveLicense` によって条件付きでロックされていたフロントエンド機能は、常にライセンスが有効であるかのように動作しなければならない (SHALL)。このフォークにはライセンスサーバーが存在せず、`hasActiveLicense` が正当に `true` になることは決してないためである。これは、テーマ/ウィンドウ透過度コントロール、応答の長さ・言語・自動スクロール設定、チャット入力エリア(添付・マイク・スクリーンショット・送信)、Pluelyプロンプトプリセット、AIプロンプト生成、サイドバーのサポートメニュー項目、スクリーンショット範囲選択キャプチャモード、音声ポップオーバー内のスクリーンショット添付ボタン、キーボードショートカットの再割り当てに適用される。

#### Scenario: ユーザーがライセンスなしでテーマと透過度を変更する
- **WHEN** ユーザーが設定画面を開き、テーマまたはウィンドウ透過度のコントロールを調整する
- **THEN** コントロールは完全に操作可能であり、変更が反映される。「ライセンスが必要です」という表示や無効化状態は出ない

#### Scenario: ユーザーが応答の長さ・言語・自動スクロールをカスタマイズする
- **WHEN** ユーザーが応答設定画面を開き、応答の長さ・応答言語・自動スクロールを変更する
- **THEN** 変更が反映される。プレミアム機能バナーや無効化されたコントロールは表示されない

#### Scenario: ユーザーがチャット入力エリアをフルに使用する
- **WHEN** ユーザーがチャット画面を開く
- **THEN** 入力エリア(添付・マイク・スクリーンショット・テキストエリア・送信ボタン)は完全に利用可能であり、「ライセンスが必要です」というオーバーレイバナーは表示されない

#### Scenario: ユーザーがPluelyプロンプトプリセットを選択、またはAIでプロンプトを生成する
- **WHEN** ユーザーがプロンプトプリセットカードまたはAIプロンプト生成をクリックする
- **THEN** ロックアイコン・暗転スタイル・「ロック解除」CTAなしに、通常どおり処理が進む

#### Scenario: ユーザーがスクリーンショットの範囲選択キャプチャモードを使う
- **WHEN** ユーザーがスクリーンショット設定を範囲選択(「範囲選択」)モードに切り替えてキャプチャする
- **THEN** 「ライセンスが必要です」というエラーなしに、通常どおりキャプチャが行われる

#### Scenario: ユーザーがキーボードショートカットを再割り当てする
- **WHEN** ユーザーがショートカット設定を開き、カスタマイズ可能なショートカット(`move_window` を除く)のキー割り当てを変更しようとする
- **THEN** 「変更」コントロールが有効になっており、ロックアイコンや「カスタマイズにはライセンスが必要です」という表示なしに、再割り当てが成功する

### Requirement: ライセンス購入・Pluelyホスト型APIへの導線はUIから削除されている
このフォークは決済バックエンドを一切設定しないため、UIは存在しないバックエンドへの行き止まりとなるライセンス購入・ライセンスキー入力の導線を表示してはならない (SHALL NOT)。これには、`GetLicense` の決済URL CTA(すべての利用箇所)、紹介・クーポン促進カード(`Promote.tsx`)、およびダッシュボードのPluely APIセットアップ画面にあるライセンスキー入力・有効化・無効化・「Pluely API 有効化」トグル・Pluelyホストモデル選択セクションが含まれる。

#### Scenario: アプリ内のどこにもライセンス購入CTAが表示されない
- **WHEN** ユーザーが設定・ダッシュボード・応答設定・システムプロンプト・ショートカット・チャット/オーバーレイUIを操作する
- **THEN** ライセンス決済フローを開始するボタンやリンクは存在せず、「有効なライセンスが必要です」という表示もされない

#### Scenario: ダッシュボードにライセンスキー入力欄がもう表示されない
- **WHEN** ユーザーがダッシュボードを開く
- **THEN** ライセンスキー入力欄・有効化/無効化コントロール・「Pluely API 有効化」トグルは存在しない

### Requirement: システムレベル(Rust/Tauri)のライセンスゲートは明示的に変更しないままにする
ライセンス状態を強制するバックエンド/システムレベルのコードは、このフォークで既に非機能であるとしても、この change によって変更されてはならない (SHALL NOT)。これには、`src-tauri/src/shortcuts.rs` の `LicenseState` による矢印キー `move_window` グローバルショートカットのゲート、および `src-tauri/src/activate.rs` と `src-tauri/src/api.rs` の該当部分にある Pluely の決済・チェックアウト・プロキシバックエンドが含まれる。

#### Scenario: 矢印キーでのウィンドウ移動ショートカットは非機能のまま
- **WHEN** ユーザーが `move_window` の矢印キーショートカットを割り当てて実行する
- **THEN** (現状の挙動から変わらず)引き続き黙って no-op になる。マウスによるウィンドウドラッグ(`personal-fork-ux` capability により別途提供される)が既に本質的なニーズをカバーしており、Rust側のゲートは意図的に手を付けないままにするため
