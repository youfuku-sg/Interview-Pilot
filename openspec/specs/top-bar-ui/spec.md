## Purpose

トップバーを Interview-Pilot の常時表示インターフェースとして構成し、音声操作、文字起こし、AI回答、テキスト入力を限られた領域内で扱えるようにする。

## Requirements

### Requirement: Top bar layout structure
トップバーは左カラム（アイコン縦並び）と右エリア（テキスト入力）の2カラム構成でなければならない（SHALL）。左カラムの幅は固定（約40px）、右エリアは残り幅をすべて占める。トップバーの collapsed 時の縦幅は約240pxとし、現状（160px）の1.5倍とする。左カラムの配置順は上から SystemAudioボタン・マイクボタン・設定ボタン・終了ボタン・ドラッグハンドルとする。

#### Scenario: 通常時（未キャプチャ）のレイアウトが2カラムである
- **WHEN** アプリが起動し、音声キャプチャが停止中の状態でトップバーが表示される
- **THEN** 左カラムにアイコンボタン群（SystemAudioボタン・マイクボタン・設定ボタン・終了ボタン・ドラッグハンドル）が縦に並んで表示される
- **AND** 右エリアにテキスト入力欄が表示される

#### Scenario: 左カラムのアイコンが正しい順序で縦に並ぶ
- **WHEN** トップバーが表示される
- **THEN** 上から順に SystemAudioボタン、マイクボタン、設定ボタン、終了ボタン、ドラッグハンドルが縦方向に配置される

#### Scenario: トップバーの collapsed 縦幅が約240pxである
- **WHEN** ポップオーバーが閉じている状態（collapsed）でトップバーが表示される
- **THEN** ウィンドウの高さが約240pxである

#### Scenario: 音声キャプチャ中も2カラム構成を維持する
- **WHEN** システム音声のキャプチャが開始される
- **THEN** 左カラムにステータス表示、右エリアにオーディオビジュアライザーが表示される
- **AND** 2カラムのレイアウト構造が維持される

#### Scenario: テキスト入力欄が右エリアの幅いっぱいに広がる
- **WHEN** トップバーが表示される
- **THEN** テキスト入力欄が右エリアの横幅を占有して表示される

#### Scenario: 終了ボタンを押すとアプリが終了する
- **WHEN** ユーザーが左カラムの終了ボタンを押す
- **THEN** アプリケーションプロセスが終了する

### Requirement: Top bar right panel layout
トップバーの右エリアは縦に3段（上段・中段・下段）に分割されなければならない（SHALL）。上段は文字起こしパネル、中段はAI回答パネル、下段はテキスト入力欄として機能する。

#### Scenario: 右エリアが3段に分割されている
- **WHEN** トップバーが表示される
- **THEN** 右エリアが上段・中段・下段の3つの横枠に縦分割されている

#### Scenario: 上段が文字起こしパネルとして機能する
- **WHEN** 音声キャプチャが有効でトップバーが表示される
- **THEN** 上段に最新の文字起こし結果が表示される

#### Scenario: 中段がAI回答パネルとして機能する
- **WHEN** 文字起こし後にAI処理が完了する
- **THEN** 中段にAIの回答テキストが表示される

#### Scenario: AIプロバイダー未設定時にエラーを表示する
- **WHEN** AIプロバイダーが設定されておらず、かつ Pluely API が無効の状態でトップバーが表示される
- **THEN** 中段に「AIプロバイダーが選択されていません」というエラーメッセージが表示される

#### Scenario: AI処理中にローディングを表示する
- **WHEN** 文字起こし完了後にAI処理（`isAIProcessing`）が実行されている
- **THEN** 中段にローディングインジケーターが表示される

#### Scenario: テキスト入力欄が右エリアの最下段に配置される
- **WHEN** トップバーが表示される
- **THEN** テキスト入力欄（`Completion` の `Input`）が右エリアの一番下の枠に表示される

#### Scenario: 上段・中段の高さが均等に分配される
- **WHEN** トップバーが表示される
- **THEN** 上段と中段がそれぞれ利用可能な残り高さの半分ずつを占める

#### Scenario: 各段の境界が視覚的に区別できる
- **WHEN** トップバーが表示される
- **THEN** 上段・中段・下段の境界線が表示され、3つのエリアが視覚的に区別できる
